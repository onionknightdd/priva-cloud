# 技能管理面板详细操作指南

点击顶部导航栏的「Skills」标签页进入技能管理。

---

```

│ [All][Proj][G]│ │SKILL.md  │ │  ---                     │
```

## 技能列表侧边栏（左栏）

### 级别筛选

顶部三个标签切换显示范围：

```
[All]  [Project]  [Global]
```

- **All**：显示所有技能
- **Project**：仅显示项目级技能
- **Global**：仅显示全局技能

### 搜索

搜索框按技能名称实时过滤（大小写不敏感）。

### 技能列表

技能按级别分组显示：

```
PROJECT ──────────────────
  ▎pdf               ← 选中状态（蓝色左边框）
    PDF 创建和处理      ← 描述文字
  docx
    Word 文档操作

GLOBAL ───────────────────
  xlsx    🟢           ← 已启用（绿色开关）
  utils   🔘           ← 已禁用（灰色开关，文字半透明）
```

- **PROJECT（项目级）**：仅当前用户可见和使用的技能
- **GLOBAL（全局级）**：所有用户共享的技能，每个用户可单独启用/禁用

### 全局技能开关

全局技能名称右侧显示开关图标：
- 🟢 **启用**（绿色）：Agent 可以使用此技能
- 🔘 **禁用**（灰色）：Agent 不会看到此技能，技能项显示为半透明
- 点击图标切换状态
- 项目级技能始终启用，无开关

### 顶部操作栏

搜索框右侧：
- **🔄 刷新**：手动刷新技能列表
- **📤 上传**：上传新技能包（详见下方）。管理员点击后显示下拉菜单选择 project / global 级别

搜索栏下方：
- **🏪 Skill Hub**：全宽按钮，打开技能市场（详见下方）

### 底部操作栏

```
┌─────────────────────────────┐
│ [⚙ 设置]            [◀ 折叠]│
└─────────────────────────────┘
```

- **设置**：打开设置弹窗
- **折叠**：折叠侧边栏为 48px 图标模式（仅显示上传、设置、展开按钮）

---

## 文件树（中栏）

### 头部

显示当前技能名称，右侧操作按钮：
- **📥 下载**：将整个技能导出为 `.tar.gz` 压缩包
- **🗑 删除**：删除技能（仅 project 级别技能或管理员可见）

### 文件搜索

顶部搜索框按文件名过滤，支持递归搜索子目录：
- 搜索时自动展开包含匹配文件的目录
- 清空搜索恢复原始树结构

### 文件树

```
📄 SKILL.md          ← 技能主文件（默认展开）
📁 scripts/          ← 点击展开/折叠
   📄 create.py
   📄 validate.py
📁 references/
   📄 api.md
📁 assets/
   📄 template.js
```

- 点击文件在右栏预览内容
- 当前选中文件蓝色高亮
- SKILL.md 默认自动选中展示
- 面板宽度可拖拽调整（180px ~ 480px）

---

## 文件预览（右栏）

### 头部

```
┌─────────────────────────────────────────┐
│  scripts/create.py        [预览] [源码] [📋] │
└─────────────────────────────────────────┘
```

- 显示完整文件路径（等宽字体）
- **预览/源码切换**（仅 Markdown 文件）：
  - **预览**：Markdown 渲染视图（格式化标题、表格、列表、代码块等）
  - **源码**：原始文本 + 语法高亮
- **📋 复制**：复制文件全部内容（二进制文件隐藏）

### 代码显示

- 语法高亮：支持 bash、python、javascript、typescript、yaml、json、xml、css、sql、go、rust、java、markdown、dockerfile 等 14+ 语言
- 行号：左侧固定列显示行号
- 等宽字体（JetBrains Mono）
- 二进制文件显示提示信息，无法预览

### 技能优化功能（仅 project 级别技能）

选中文件中的代码后，自动在鼠标位置弹出 Optimize 气泡：

```
操作流程：
1. 在源码视图中用鼠标选中一段代码
2. 松开鼠标后，选中区域附近弹出气泡：
   ┌──────────────┐
   │ ✨ Optimize   │
   │ L12-L25       │  ← 显示选中行号范围
   └──────────────┘

3. 点击气泡，打开优化弹窗：
   ┌─────────────────────────────────────┐
   │ ✨ Optimize Selection          [×]  │  ← 可拖拽标题栏
   │                                     │
   │ ┌─────────────────────────────────┐ │
   │ │ > def calculate():              │ │  ← 选中的代码 +2 行上下文
   │ │ >     result = 0                │ │
   │ │ >     for i in range(n):        │ │
   │ │ >         result += i           │ │
   │ └─────────────────────────────────┘ │
   │                                     │
   │ Your comments:                      │
   │ ┌─────────────────────────────────┐ │
   │ │ 请优化这段代码的性能...           │ │
   │ └─────────────────────────────────┘ │
   │                                     │
   │                          [📤 Send]  │
   └─────────────────────────────────────┘  ← 右下角可拖拽调整大小

4. 输入优化建议，按 Enter 或点击 Send
5. 系统自动：
   - 清除当前聊天
   - 切换到 Priva 聊天标签页
   - 组装消息（包含技能名、文件路径、行号、代码、评论）
   - 自动发送给 Agent
```

- 弹窗可拖拽移动（标题栏拖拽）和缩放（右下角拖拽）
- 按 **Escape** 关闭弹窗
- 按 **Enter** 发送（Shift+Enter 换行）
- 全局技能不显示优化功能

## 上传新技能

### 操作步骤

**普通用户：**
1. 点击左栏底部「上传技能」按钮
2. 选择技能包文件
3. 自动上传到 **project** 级别

**管理员：**
1. 点击「上传技能」按钮旁的下拉箭头
2. 选择目标级别：
   - **Upload skill**：上传为项目级技能
   - **Upload global skill**：上传为全局技能
3. 选择技能包文件

### 支持的文件格式

- `.zip`
- `.tar`
- `.tar.gz`
- `.tgz`
- `.skill`

### 技能包结构要求

```
my-skill/
├── SKILL.md           ← 必需！技能主文件
├── scripts/           ← 可选，辅助脚本
│   └── helper.py
├── references/        ← 可选，参考文档
│   └── guide.md
└── assets/            ← 可选，模板资源
    └── template.docx
```

### SKILL.md 格式

```markdown
---
name: my-skill
description: 技能描述，说明此技能做什么以及何时使用
metadata:
  icon: FileText         # 可选，Lucide 图标名称
  icon_color: "#58a6ff"  # 可选，图标颜色
---

# 技能标题

技能的详细使用说明...
```

- `name`：技能名称（小写字母、数字、连字符）
- `description`：技能描述（用于 Agent 判断何时调用此技能）
- `metadata`：可选元数据（图标、颜色等）

### 常见上传错误

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| 缺少 SKILL.md | 包内没有 SKILL.md 文件 | 确保根目录包含 SKILL.md |
| 前置元数据无效 | name 或 description 缺失/格式错误 | 检查 YAML 前置元数据 |
| 同名技能已存在 | 已有同名技能 | 先删除旧技能再上传 |

## 删除技能

1. 在文件树头部点击 🗑 删除按钮
2. 弹出确认对话框，**需要手动输入技能名称**才能删除：

```
┌─────────────────────────────────────────┐
│ Delete Skill                             │
│                                         │
│ Delete skill "my-skill"?                │
│ This action cannot be undone.            │
│                                         │
│ Type the skill name to confirm:          │
│ ┌─────────────────────────────────────┐ │
│ │ my-skill                             │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ [Cancel]                      [Delete]  │
└─────────────────────────────────────────┘
```

3. 输入正确的技能名后「Delete」按钮变为可点击
4. 点击「Delete」确认删除

**权限：** 项目级技能可由所有者删除，全局技能仅管理员可删除。

2. 技能打包为 `.tar.gz` 文件自动下载
3. 文件名格式：`技能名.tar.gz`

## 在聊天中使用技能

### 方式一：`/` 命令

1. 在聊天输入框输入 `/`
2. 弹出技能选择器（分 PROJECT / GLOBAL 组）
3. 选择目标技能（显示为输入框内标签）
4. 输入你的具体指令
5. 发送

```
/pdf 请将这份报告转换为 PDF 格式
/docx 创建一份包含以下内容的会议纪要
/xlsx 分析这个表格中的销售数据
```

### 方式二：自然对话

直接描述需求，Agent 根据技能描述自动判断是否调用：

```
"帮我把这个 Word 文档转成 PDF"
→ Agent 自动识别并使用 pdf 技能
```

## 技能层级说明

| 级别 | 可见范围 | 管理权限 | 启用/禁用 |
|------|---------|---------|----------|
| **PROJECT** | 仅当前用户 | 用户自行上传/删除 | 始终启用 |
| **GLOBAL** | 所有用户共享 | 管理员管理 | 每个用户可单独开关 |

- 内置技能（如 pdf、docx、xlsx）属于 GLOBAL 级别
- 禁用全局技能后，Agent 在该用户的会话中不会看到和使用此技能
