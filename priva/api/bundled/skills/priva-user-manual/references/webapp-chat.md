# 聊天界面详细操作指南

## 输入框功能

### 文本输入

- 在输入框中输入文字，按 **Enter** 发送
- 支持多行输入（Shift + Enter 换行）
- 输入框自动根据内容高度调整，最大 200px

### 技能调用 (`/`)

```
操作流程：
1. 输入 /
2. 弹出技能选择面板
   ┌───────────────────────────────────┐
   │ Select an agent skill             │
   │ ┌─────────────────────────────┐  │
   │ │ PROJECT                      │  │
   │ │  ▸ pdf                       │  │
   │ │    PDF 创建和处理             │  │
   │ │  ▸ docx                      │  │
   │ │    Word 文档操作              │  │
   │ │ GLOBAL                       │  │
   │ │  ▸ xlsx                      │  │
   │ │    Excel 操作                │  │
   │ └─────────────────────────────┘  │
   └───────────────────────────────────┘
3. 点击或方向键选择技能
4. 技能以标签形式显示在输入框内
   ┌────────────────────────────────────────────┐
   │ ▸ skill: pdf  PROJECT  ×                   │
   │                                            │
   │ 请帮我把这个表格转成 PDF                      │
   │                                            │
   │ [+] [🛡 Default]            [模型▼] [发送]  │
   └────────────────────────────────────────────┘
5. 输入你的具体指令
6. 发送
```

- 输入 `/` 后继续打字可过滤技能列表
- 按 Escape 关闭选择器
- 按 **Arrow Up/Down** 在列表中导航
- 按 **Enter** 或 **Tab** 选中当前高亮技能
- 选中后技能显示为标签（chip），带有名称、级别（PROJECT/GLOBAL）和 × 关闭按钮
- 输入框为空时按 **Backspace** 也可移除已选技能标签

### 文件引用 (`@`)

```
操作流程：
1. 输入 @
2. 弹出已上传文件列表
   ┌─────────────────────────────────────┐
   │ Select an uploaded file              │
   │ ┌─────────────────────────────────┐ │
   │ │ 📄 report.pdf   1.2MB  03/25   │ │
   │ │ 📄 data.xlsx    340KB  03/25   │ │
   │ │ 📄 photo.jpg    2.1MB  03/24   │ │
   │ └─────────────────────────────────┘ │
   └─────────────────────────────────────┘
3. 点击选择文件
4. 文件作为附件添加到输入框
5. 可继续输入指令，然后发送
```

- 输入 `@` 后继续打字可按文件名过滤
- 按 **Arrow Up/Down** 在列表中导航
- 按 **Enter** 或 **Tab** 选中当前高亮文件
- 按 Escape 关闭选择器
- 选中的文件显示为附件标签（chip），带有文件名、大小和 × 关闭按钮

### 文件上传

**方式一：拖拽上传**
- 将文件直接拖到输入框区域
- 出现「Drop files here」蓝色提示时松手

**方式二：点击上传**
- 点击输入框左下角的 **+** 按钮
- 在弹出菜单中选择「Upload file」
- 从文件选择器中选择文件

**方式三：粘贴图片**
- 直接 Ctrl+V / Cmd+V 粘贴剪贴板中的图片
- 图片自动压缩处理

**限制：**
- 单个文件最大 3MB
- 单次最多 5 个文件
- 支持格式：docx、xlsx、pptx、pdf、txt、csv、json、md、py、js、ts、html、css、sh、sql 等代码文件
- 图片格式：png、jpg、jpeg、gif、webp

**上传后：**
- **普通文件**：显示为带左侧青色边框的标签，包含文件名、大小和 × 移除按钮
  - 上传中显示黄色加载动画，失败显示红色
- **图片文件**：显示为 64×64 缩略图预览，右上角 × 可移除
  - 处理中显示半透明加载遮罩
- 如果配置了 vision 模型，附加图片时会显示提示：「Using vision model: xxx」
- 如果未配置 vision 模型，会显示警告提示

## + 按钮菜单

输入框左下角的 + 按钮，点击展开菜单：

```
┌─────────────────────────┐
│ 📤 Upload file           │
│ ─────────────────────── │
│ 🔌 MCP               ▸ │ ← 悬浮展开子菜单
└─────────────────────────┘
```

### MCP 子菜单

```
┌─────────────────────────┐
│ [Auto] [Disable]   2/3  │
│ ─────────────────────── │
│ ☑ my-search       HTTP  │
│ ☑ my-tools        HTTP  │
│ ☐ test-server     HTTP  │
└─────────────────────────┘
```

- **Auto**（默认）：所有 MCP 服务器自动可用，列表显示为全选状态
- **Disable**：禁用所有 MCP 工具，列表变暗
- **自选模式**：点击任意服务器复选框切换到自选模式，显示选中数/总数
- 每个服务器显示名称和类型（HTTP）

## 权限模式详解

输入框工具栏左侧的 🛡 权限按钮：

```
┌────────────────────────────────┐
│ 🛡 Default                     │ ← 点击展开
├────────────────────────────────┤
│▎Bypass                         │
│  所有操作自动执行                │
│                                │
│ Default                        │ ← 当前选中
│  危险操作前询问                  │
│                                │
│ Accept Edits                   │
│  自动批准文件编辑                │
│                                │
│ Plan                           │
│  先生成计划，确认后执行           │
└────────────────────────────────┘
```

### Bypass 模式
- Agent 拥有完全自主权，自动执行所有操作
- **适用场景**：你完全信任 Agent 的行为，不需要监督
- **风险**：Agent 可能执行危险的 bash 命令或修改文件
- 按钮显示黄色

### Default 模式（推荐）
- Agent 执行危险操作前会弹出权限请求卡片
- 按钮显示绿色

### Accept Edits 模式
- 文件读取和编辑操作自动批准
- 其他危险操作（如 bash 命令）仍会询问
- **适用场景**：让 Agent 自由编辑代码，但控制系统命令
- 按钮显示紫色

### Plan 模式
- Agent 先生成完整计划，你审查后再执行
- 计划显示在 Canvas 面板的 PLAN 标签页
- **适用场景**：复杂任务，需要先确认方案
- 按钮显示青色

## Canvas 面板

### 自动弹出时机
- Agent 开始执行工具调用时自动展开
- 加载历史会话时，如有任务或文件操作也会展开

### TASKS 标签页
```
┌─────────────────────────┐
│ TASKS                    │
│ ▰▰▰▰▱▱▱ 3/7            │ ← 进度条
│ ├─ 分析需求        ✅    │
│ ├─ 编写代码              │
│ │  ├─ 创建文件     🟣    │
│ │  └─ 编辑配置     ⏳    │
│ └─ 运行测试        ⏳    │
│                         │
│ ☑ 创建 README      完成  │ ← 待办列表
│ ☐ 更新 CHANGELOG        │
└─────────────────────────┘
```
- 顶部进度条显示完成比例
- 树状结构展示任务层级
- 待办列表（TodoList）展示简单的检查项
- 左侧彩色边框表示状态：紫色=进行中，绿色=完成，红色=失败，黄色=等待

### FILES 标签页
```
┌──────────────────────────────────┐
│ FILES                             │
│ Turn 1                            │
│ ┌──────────────────────────────┐ │
│ │ WRITE src/utils/helper.py    │ │
│ │ + def calculate():           │ │
│ │ +     return result          │ │
│ └──────────────────────────────┘ │
│ Turn 2                            │
│ ┌──────────────────────────────┐ │
│ │ EDIT src/main.py             │ │
│ │ - old_code()                 │ │
│ │ + new_code()                 │ │
│ └──────────────────────────────┘ │
└──────────────────────────────────┘
```
- 按轮次分组显示文件操作
- WRITE = 新建文件，EDIT = 修改文件
- 展示 diff 预览（绿色=新增，红色=删除）

### PLAN 标签页
- 显示 Agent 生成的执行计划（Markdown 渲染）
- 仅在 Plan 权限模式下使用

### 折叠/最小化
- 点击「Minimize」收缩为 40px 窄条
- 最小化状态显示完成进度（如 `3/5`）、文件操作数（如 `3F`）或 `PLAN`
- 有正在运行的任务时紫色边框脉冲闪烁提示
- 可拖拽左边缘调整面板宽度（最小 280px，最大 60% 窗口宽度）

## Agent 交互卡片

### 权限请求
Agent 需要执行危险操作时弹出，需要你批准或拒绝：

```
┌────────────────────────────────────┐
│ ⚠ PERMISSION REQUEST               │
│                                    │
│ Tool: 💻 Bash                      │
│ ┌────────────────────────────────┐ │
│ │ $ rm -rf /tmp/old_data         │ │
│ └────────────────────────────────┘ │
│                                    │
│              [Deny]  [Allow]       │
└────────────────────────────────────┘
```

- 显示工具名称和具体操作内容
- **Bash** 命令：显示命令行预览
- **Write** 操作：显示文件路径和内容预览（截断到 200 字符）
- **Edit** 操作：显示文件路径和 diff（红色删除行 + 绿色新增行）
- **Read** 操作：显示文件路径
- 其他工具：显示 JSON 格式参数
- 批准后折叠为绿色单行摘要，拒绝后折叠为红色单行摘要
- **危险命令**（包含 rm、sudo、chmod 等）：点击 Allow 后额外弹出确认对话框

### 用户提问
Agent 需要你提供额外信息时弹出：
```
┌────────────────────────────────────┐
│ 🤔 Priva needs your feedback       │
│                                    │
│ 你希望使用哪个数据库？              │
│   ○ PostgreSQL                     │
│     适合大型应用                    │
│   ○ MySQL                          │
│     轻量级选择                      │
│   ○ SQLite                         │
│     本地开发用                      │
│                                    │
│   Type a custom answer...          │
│                                    │
│                 [▸ Send Answer]     │
└────────────────────────────────────┘
```
- 支持**单选**和**多选**两种模式
- 每个选项显示标签和可选描述
- 点击「Type a custom answer」可展开自定义文本输入框
- 单选模式下选择自定义输入会清除选项选择，反之亦然
- 回答完成后点击「Send Answer」提交
- 「跳过」可以忽略问题，直接在输入框发消息

### 计划审批
权限模式为 Plan 时，Agent 生成计划后弹出：
```
┌────────────────────────────────────┐
│ 📋 PLAN READY                      │
│                                    │
│ Agent 的计划已就绪。View plan       │
│                                    │
│ ○ 批准并开始（新会话 + 自动编辑）    │
│   清除当前会话，新会话中自动批准编辑  │
│ ○ 批准并开始（当前会话 + 自动编辑）  │
│   在当前会话继续，自动批准编辑       │
│ ○ 批准并开始（手动批准编辑）         │
│   在当前会话继续，每次编辑需手动确认  │
│ ○ 提供反馈                          │
│   输入修改意见，Agent 重新调整计划    │
│                                    │
│                    [▸ Confirm]      │
└────────────────────────────────────┘
```
- 点击「View plan」可跳转到 Canvas 面板 PLAN 标签页查看详细计划
- 选择「提供反馈」后出现文本输入框
- 选择其中一个选项后点击「Confirm」执行

### 工具调用展示
```
┌────────────────────────────────────┐
│ ▶ 3 tool steps                     │ ← 点击展开
├────────────────────────────────────┤
│ ✅ Read src/main.py                │
│ 🟣 Bash: npm test          RUNNING │
│ ⏳ Write src/output.txt            │
└────────────────────────────────────┘
```
- 默认折叠，显示工具步骤数量
- 展开后每步显示：工具名、输入参数、输出结果、状态
- Write/Edit 操作不在消息流中显示，而是在 Canvas FILES 标签页
- Agent/Task 等后台工具也不在消息流中显示，而是在 Canvas TASKS 标签页

## 引用回复

可以引用消息中的文字进行回复：

```
┌────────────────────────────────────────────┐
│ ↵ QUOTED                                    │
│ "这段代码有性能问题..."                ×     │
│                                            │
│ 请详细解释一下这个问题                       │
│                                            │
│ [+] [🛡 Default]            [模型▼] [发送]  │
└────────────────────────────────────────────┘
```

- 引用文本显示在输入框上方，带蓝色左边框
- 点击 × 或输入框为空时按 Backspace 可清除引用
- 发送时自动将引用和回复组合为完整消息

## 快捷操作

输入框下方显示预设的快捷操作按钮（仅在欢迎页面/无消息时显示）：

```
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│ ⚡ 生成报告   │ │ 🔍 代码审查   │ │ 📝 写文档    │ │ 💻 分析日志   │
│ 帮我生成...  │ │ 审查这段...  │ │ 创建一份...  │ │ 分析最近...  │
└──────────────┘ └──────────────┘ └──────────────┘ └──────────────┘
```

- 4 列网格布局，最多显示 12 个快捷操作
- 每个按钮显示图标、名称和提示词预览
- 点击按钮，提示词模板填入输入框并进入变量模式
- 变量模式下：包含 `{变量}` 时自动选中第一个变量，按 **Tab** 跳转到下一个变量
- 按 **Escape** 退出变量模式并清空输入
- 在「设置」中添加/编辑/删除快捷操作

## 侧边栏

### 会话列表
- 按时间分组：**Today** / **Yesterday** / **More**（更早）
- 更早的会话默认折叠，点击「Show more」展开
- 当前活跃会话高亮显示（蓝色左边框 + 背景色）
- 显示会话 ID 或自动生成的名称

### 来源筛选
```
┌───────────────────────────┐
│ [项目] [全局] [全部]  🔄    │
└───────────────────────────┘
```
- **项目**：仅显示当前项目的会话
- **全局**：仅显示全局会话
- **全部**：显示所有会话
- 右侧刷新按钮手动刷新列表

### 操作
- **新对话**：顶部「New Chat」按钮
- **搜索**：搜索框，按 ID 或名称过滤
- **删除**：悬浮会话项显示 ··· 菜单 → 选择「Delete」→ 确认对话框
- **加载更多**：滚动到底部自动触发，或点击「Load more」按钮

### 调整宽度
- 拖拽右侧边缘调整
- 点击底部折叠按钮折叠为 48px（仅图标）
- 宽度设置自动保存

### 底部栏
- **设置按钮**：点击打开设置弹窗（SettingsPopover）
- **折叠按钮**：折叠/展开侧边栏
- 折叠状态下仅显示新对话图标按钮和设置/展开按钮
