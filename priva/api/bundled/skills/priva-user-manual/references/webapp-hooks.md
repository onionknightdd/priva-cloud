## 生命周期图谱（中栏）

### 交互式 SVG 可视化

展示所有 Hook 之间的执行流程关系：

- **Hook 节点**：圆角方框，边框颜色按阶段区分（绿=Session，蓝=Tool，紫=Agent，黄=Misc）
  - 显示 Hook ID（等宽字体）
  - 已配置的显示 `configured: N`
  - 🔴 红点标记可拦截事件
  - 点击选中（加粗边框）→ 弹出说明气泡 + 打开右侧详情
- **连线**：带箭头的曲线，显示执行顺序（实线=常规流程，虚线=条件流程）
- **装饰节点**：虚线框标注 Agent Process / Tool Executes / Subagent Works 等阶段
- **图例**：右下角显示阶段颜色和 can-block 标记说明

### 操作

| 操作 | 方式 |
|------|------|
| 缩放 | 鼠标滚轮（0.4x ~ 3x） |

| 选中节点 | 点击 Hook 方框 |
| 取消选中 | 点击背景空白处 |
| 重置视图 | 双击背景 或 缩放控件「100%」按钮 |

### 缩放控件（右下角覆盖层）

```
[−]  [100%]  [+]  [Fit]
```

### 节点弹窗

- Hook ID + 阶段标签 + BLOCK 标签

- 使用说明（格式化键值对）
- 关闭按钮

### 执行顺序说明

1. **Setup** → Agent 进程启动时运行一次（仅一次）
2. **SessionStart** → 每个会话开始时触发（包括启动、恢复、清除、压缩后重启）
3. **UserPromptSubmit** → 用户消息提交后、Agent 处理前，可修改或拦截输入
4. Agent 开始处理，根据需要分支进入 **Tool Pipeline** 和/或 **Agent Pipeline**：
   - **Tool Pipeline**：PreToolUse → PermissionRequest → 工具执行 → PostToolUse / PostToolUseFailure
   - **Agent Pipeline**：SubagentStart → 子代理执行 → SubagentStop
5. **Stop** → Agent 完成或收到停止信号时触发，可拦截以保持运行
6. **PreCompact** → 上下文窗口即将压缩前触发
7. **SessionEnd** → 会话结束时触发
8. **Notification** → 在任何阶段都可能触发的通知事件

## 生命周期事件详解

### SESSION 组（绿色）

#### Setup
- **触发时机**：Agent 进程启动时运行一次，在任何会话开始之前
- **可拦截**：否
- **Matcher**：无
- **典型用途**：初始化环境 — 从保险库加载 API Key、设置日志、配置数据库连接。确保所有必要资源在 Agent 处理请求前可用

#### SessionStart
- **触发时机**：每次会话开始时（启动、恢复、清除、压缩后重启）
- **可拦截**：否
- **Matcher**：`source`（startup | resume | clear | compact）
- **典型用途**：响应新会话或恢复的会话 — 加载用户偏好、恢复对话上下文、跟踪会话分析。根据启动方式个性化行为

#### SessionEnd
- **触发时机**：会话结束时（清除、登出、进程退出）
- **可拦截**：否
- **Matcher**：`reason`（clear | logout | ...）
- **典型用途**：会话清理 — 刷新日志、保存会话状态、释放临时文件、通知监控服务。确保无论退出原因如何都能正确清理资源

### TOOL 组（蓝色）

#### UserPromptSubmit 🔴
- **触发时机**：用户输入提交后、Agent 处理前
- **可拦截**：**是** — 可阻止消息发送给 Agent
- **Matcher**：无
- **典型用途**：输入验证和转换 — 从提示中去除 PII、执行内容策略、注入额外上下文。添加合规层，在消息到达模型前进行脱敏

#### PreToolUse 🔴
- **触发时机**：工具执行前
- **可拦截**：**是** — 可修改参数或阻止调用
- **Matcher**：`tool name`（如 Bash、Edit|Write、mcp__*）
- **典型用途**：拦截工具调用 — 阻止写入受保护目录、添加审计日志、按环境重定向路径。安全策略执行，如阻止包含 `rm -rf` 或 `sudo` 的 Bash 命令

#### PermissionRequest 🔴
- **触发时机**：工具需要显式权限时
- **可拦截**：**是** — 可自动批准或拒绝
- **Matcher**：`tool name`
- **典型用途**：基于规则的权限管理 — 自动允许只读文件操作、要求网络调用审批。构建细粒度权限系统，用策略规则替代手动审批

#### PostToolUse
- **触发时机**：工具执行成功后
- **可拦截**：否
- **Matcher**：`tool name`
- **典型用途**：处理工具结果 — 解析输出中的指标、缓存结果、触发后续操作。收集使用统计，记录每次工具调用的耗时、输入大小和输出

#### PostToolUseFailure
- **触发时机**：工具执行失败后
- **可拦截**：否
- **Matcher**：`tool name`
- **典型用途**：处理工具失败 — 记录错误详情、触发告警、提供备用逻辑

### AGENT 组（紫色）

#### SubagentStart
- **触发时机**：子代理被创建时
- **可拦截**：否
- **Matcher**：`agent type`
- **典型用途**：配置或跟踪子代理 — 分配资源限制、注入专用系统提示、记录生成用于成本追踪。按代理类型应用预算或添加分布式追踪头

#### SubagentStop
- **触发时机**：子代理完成或被停止时
- **可拦截**：否
- **Matcher**：`agent type`
- **典型用途**：处理子代理结果 — 汇总多个子代理输出、释放分配的资源、更新进度。将并行研究子代理的结果合并为统一回复

#### Stop 🔴
- **触发时机**：Agent 收到停止信号时
- **可拦截**：**是** — 可阻止 Agent 关闭
- **Matcher**：无
- **典型用途**：拦截 Agent 关闭 — 确保待处理操作完成、提示确认、自动保存进行中的工作。防止过早关闭，让 Agent 继续运行直到所有排队任务完成

### MISC 组（黄色）

#### Notification
- **触发时机**：Agent 发送通知时（随时可能）
- **可拦截**：否
- **Matcher**：`notification type`
- **典型用途**：自定义通知投递 — 根据严重性或内容路由到 Slack、邮件或 Webhook。高优先级通知发送到 PagerDuty，信息级通知发送到 Slack 频道

#### PreCompact
- **触发时机**：对话历史即将被压缩前
- **可拦截**：否
- **Matcher**：`trigger`（manual | auto）
- **典型用途**：压缩前保存数据 — 将重要事实提取到外部记忆、决定保留哪些上下文。在关键决策和代码引用被压缩前保存到知识库

### 处理器类型

#### Command（Shell 脚本）

```
COMMAND
┌──────────────────────────────────────┐
│ .claude/hooks/my-hook.sh             │
└──────────────────────────────────────┘
```

- 执行一条 shell 命令
- 退出码 0 = 通过，非 0 = 拦截（对可拦截事件）
- stdout/stderr 作为结果返回
- 可查看脚本文件内容（ScriptContentLoader）

#### HTTP（请求 URL）

```
URL
┌──────────────────────────────────────┐
│ https://example.com/hook             │
└──────────────────────────────────────┘

HEADERS
┌──────────┬────────────────────────┐
│ Auth     │ Bearer xxx          [×]│
└──────────┴────────────────────────┘
[+ Add Header]
```

- 发送 HTTP POST 请求，event 数据作为 body
- 响应 JSON 中的 `decision` 字段控制行为
- 支持自定义请求头

#### Prompt（调用模型）

```
PROMPT
┌──────────────────────────────────────┐
│ 检查此操作是否安全，如不安全请拒绝...    │
└──────────────────────────────────────┘

MODEL
┌──────────────────────────────────────┐
│ claude-sonnet-4-6                    │
└──────────────────────────────────────┘
```

- 让指定模型评估事件并做出决策
- 适合复杂的安全审查逻辑

#### Agent（即将推出）

- 类型按钮显示 SOON 标签，暂不可选

## LOGS 标签页 — 执行日志

### 筛选与刷新

```
[All Events ▼]   [Auto ☐]   [🔄]
```

- **事件筛选**：下拉选择特定事件类型过滤

## 内置 Hook 管理

### 启用/禁用

每个内置 Hook 卡片右侧有操作按钮：

- **ENABLE**（蓝色）：启用此 Hook
- **DISABLE**（绿色，悬浮变红）：禁用此 Hook
- **ENFORCED**（黄色锁图标）：管理员强制启用，普通用户无法禁用

### 查看源码

内置 Hook 卡片可展开查看 Python 脚本源码：
- 行号 + 语法高亮
- 复制按钮

## 管理员强制执行

- 处理器卡片也显示锁图标，不可编辑/删除
