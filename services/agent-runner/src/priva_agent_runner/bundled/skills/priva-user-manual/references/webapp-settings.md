## API Key

用于外部程序通过 API 调用 Priva 的鉴权密钥。

```
┌──────────────────────────────────────────┐
│ Your API Key                             │
│ ┌──────────────────────────────────────┐ │
│ │ sk-●●●●●●●●●●●●●●●●     [👁] [📋]  │ │  ← 密码遮罩 + 显示切换 + 复制
│ └──────────────────────────────────────┘ │
│                                          │
│ [Regenerate]              [Revoke]       │  ← 已有 Key 时显示
│ — 或 —                                   │
│ [Generate Key]                           │  ← 无 Key 时显示
└──────────────────────────────────────────┘
```

- **Generate Key**：首次生成，显示完整 Key
- **Regenerate**：重新生成（需确认），旧 Key 立即失效
- **Revoke**：撤销 Key（需确认），永久失效
- 使用方式：API 请求头 `Authorization: Bearer <Key>`

### API 连接

```
┌──────────────────────────────────────────┐
│ BASE URL                                 │
│ ┌──────────────────────────────────────┐ │
│ │ http://your-api-server:port/         │ │
│ └──────────────────────────────────────┘ │
│ AUTH TOKEN                               │
│ ┌────────────────────────────────┬───┐  │
│ │ sk-●●●●●●●●●●●●               │👁│  │  ← 密码切换
│ └────────────────────────────────┴───┘  │
│                                          │
│ ┌─ ✅ Connected — 12 model(s) ────────┐ │  ← 自动测试结果
│ └──────────────────────────────────────┘ │
│                                  [Save]  │
└──────────────────────────────────────────┘
```

- 「Save」仅在有修改时可点击

## Channels

### WeCom 标签页

连接企业微信机器人，让用户通过企业微信与 AI Agent 对话。

#### 连接状态栏（置顶）

```
┌──────────────────────────────────────────────────┐
│▎🟢 CONNECTED    Sessions: 2   Messages: 45      │
│                          [Reconnect] [Disconnect]│
│                                                  │
│ Active Sessions:                                 │
│ session_abc123...  [📋]   user: wx_user1         │
└──────────────────────────────────────────────────┘
```

状态：
| 状态 | 颜色 | 说明 |
|------|------|------|
| CONNECTED | 🟢 绿色 | 已连接，显示活跃会话数和消息数 |
| CONNECTING | 🟡 黄色 | 正在连接 |
| AUTH FAILED | 🔴 红色 | 认证失败，检查 Bot ID 和 Secret |
| ERROR | 🔴 红色 | 错误，显示错误信息 |
| DISCONNECTED | ⬜ 灰色 | 已断开，显示「Connect」按钮 |

状态每 5 秒自动轮询刷新。

#### 配置表单

| 字段 | 说明 | 类型 |
|------|------|------|
| **Bot ID** | 企业微信机器人 ID | 等宽文本 |
| **Secret** | 机器人密钥 | 密码（可切换显示） |
| **WS Proxy URL** | WebSocket 代理地址（可选） | 等宽文本 |
| **Allowed User IDs** | 允许使用的企业微信用户 ID | 标签列表，逐个添加/删除 |
| **Welcome Message** | 新用户首次消息的欢迎语 | 多行文本 |
| **Reject Message** | 非白名单用户收到的提示 | 单行文本 |
| **Model** | 指定 WeCom 对话模型（可选） | 单行文本 |
| **Max Queue Size** | 最大排队消息数 | 数字（1-10，默认 3） |
| **Session Timeout** | 会话超时时间 | 数字（5-1440 分钟，默认 60） |

- 允许用户 ID 为空时显示黄色警告
- 修改后「Save」按钮变蓝色，点击保存

## Advanced

### 传输模式

```
[WebSocket (WS)]  [Server-Sent Events (SSE)]
```

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **WebSocket** | 双向实时通信（推荐） | 默认选择，支持权限交互 |
| **SSE** | 单向服务端推送 | WebSocket 被网络策略阻止时 |

- 选择后持久化到 localStorage
- 对话无法流式传输时可尝试切换为 SSE

## Runtime / 系统提示词（管理员）

### 系统提示词

```
┌──────────────────────────────────────────┐
│ System Prompt            [ON]  [OFF]     │
│                                          │
│ CONTENT             [EDIT]  [PREVIEW]    │
│ ┌──────────────────────────────────────┐ │
│ │ 你是一个专业的技术助手。              │ │
│ │ 请遵循以下规则：                      │ │
│ │ 1. 回答要简洁准确                    │ │
│ │ 2. 优先使用中文                      │ │
│ └──────────────────────────────────────┘ │
│ 256 characters                           │
│                                  [Save]  │
└──────────────────────────────────────────┘
```

- **ON/OFF**：全局开关
- **Edit/Preview**：编辑模式（等宽文本框）/ 预览模式（只读）
- 字符计数实时显示
- 启用后附加到所有用户的 Agent 对话中

```
│ 7    │ days     ← 0 表示不保留
└──────┘
                                   [Save]
```


### 工具重试

配置工具执行失败时的自动重试策略和回调通知。

**可重试工具列表：**
```
┌──────────────────────────────────────────┐
│ Bash                    3x / 30s    [🗑] │
│ WebFetch                2x / 60s    [🗑] │
└──────────────────────────────────────────┘

添加工具：
┌──────────────┐ ┌────────┐ ┌────────┐
│ Tool name    │ │ 3 次    │ │ 30 秒  │  [Add]
└──────────────┘ └────────┘ └────────┘
```

**回调类型：**
```
Callback: [None ▼]
```

| 类型 | 配置字段 |
|------|---------|
| **None** | 无回调 |
| **Script** | 脚本路径（等宽） |
| **WeCom** | API URL + Key（密码）+ Service Name |

每部分有独立的「Save」按钮。

## Plugins（管理员）

管理 MCP 插件配置。

```
┌──────────────────────────────────────────┐
│ my-plugin              ON  [●────]       │
│ 插件描述文字                              │
│                                          │
│ ▸ CONFIG                                 │  ← 启用后可展开
│ ┌──────────────────────────────────────┐ │
│ │ Provider Type: [MCP ▼]               │ │
│ │ MCP URL:       [https://...       ]  │ │
│ │ Tool Name:     [my-tool           ]  │ │
│ │ Timeout:       [10             ] s   │ │
│ │ Headers:                             │ │
│ │ ┌────────┬───────────────────┐       │ │
│ │ │ Auth   │ Bearer xxx     [×]│       │ │
│ │ └────────┴───────────────────┘       │ │
│ │ [+ Add Header]                       │ │
│ └──────────────────────────────────────┘ │
│                                  [Save]  │
└──────────────────────────────────────────┘
```

- 开关切换启用/禁用（立即生效）
- CONFIG 区域仅在启用后可展开
- 支持自定义请求头（添加/删除键值对）
- 每个插件独立保存
