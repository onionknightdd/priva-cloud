## 服务器列表侧边栏（左栏）

### 搜索与添加

```
┌──────────────────────────┐
│ 🔍 搜索服务器...      [+] │  ← 搜索框 + 添加按钮
│ [All] [Project] [Global] │  ← 级别筛选
└──────────────────────────┘
```

- **搜索**：按服务器名称实时过滤
- **+** 添加按钮：
  - 普通用户：直接打开添加对话框（project 级别）
  - 管理员：下拉菜单选择 Add Project Server / Add Global Server（绿色文字）
- **级别筛选**：All / Project / Global 切换

### 服务器列表

```
PROJECT ──────────────────
  ▎api-server     HTTP    ← 选中（蓝色左边框），类型标签（青色=HTTP，紫色=SSE）
   search-tools   SSE

GLOBAL ───────────────────
   shared-docs    HTTP
```

### 底部操作栏

- **设置**：打开设置弹窗
- **折叠**：折叠为 48px 图标模式

## 添加 MCP 服务器

点击 + 按钮后弹出对话框：

```
┌──────────────────────────────────────┐
│ Add Server                      [×] │
│                                      │
│ Name                                 │
│ ┌──────────────────────────────────┐ │
│ │ my-server                        │ │  ← 编辑模式下不可修改
│ └──────────────────────────────────┘ │
│                                      │
│ Type                                 │
│ ┌──────────────────────────────────┐ │
│ │ http                          ▼  │ │  ← HTTP / SSE
│ └──────────────────────────────────┘ │
│                                      │
│ URL                                  │
│ ┌──────────────────────────────────┐ │
│ │ http://localhost:3000/mcp        │ │
│ └──────────────────────────────────┘ │
│                                      │
│ Timeout                              │
│ ┌──────────────────────────────────┐ │
│ │ 60                        seconds│ │  ← 5~600 秒
│ └──────────────────────────────────┘ │
│                                      │
│ Headers                              │
│ ┌──────────┬───────────────────────┐ │
│ │ Auth     │ Bearer sk-xxx      [×]│ │
│ └──────────┴───────────────────────┘ │
│ [+ Add header]                       │
│                                      │
│ ┌─ ✅ Connected — 5 tools,         ─┐│  ← 测试结果
│ │    2 prompts, 1 resource           ││
│ └────────────────────────────────────┘│
│                                      │
│ [Cancel]     [Test]   [Add Server]   │
└──────────────────────────────────────┘
```

### 操作流程

1. 填写 Name、Type、URL（必填）
2. 可选设置 Timeout 和 Headers
3. 点击「**Test**」测试连接（**必须测试通过才能保存**）
   - 成功：绿色边框显示工具/提示词/资源数量
   - 失败：红色边框显示错误信息
4. 测试通过后「Add Server」按钮出现，点击保存

### 配置字段

| 字段 | 说明 | 要求 |
|------|------|------|
| **Name** | 服务器唯一标识 | 必填，编辑模式不可改 |
| **Type** | 连接类型 | HTTP 或 SSE |
| **URL** | MCP 服务器地址 | 必填 |
| **Timeout** | 连接超时 | 5-600 秒，默认 60 |
| **Headers** | 请求头键值对 | 可选，可添加多个 |

## 服务器元信息面板（左中栏）

选中服务器后显示元信息，面板宽度可拖拽调整（240px ~ 480px）。

### 头部

```
┌───────────────────────┐
│ my-server         [✏] │  ← 编辑按钮（project 级别或管理员可见）
└───────────────────────┘
```

### 信息展示

从上到下依次显示：

1. **类型** — 标签：`HTTP`（青色）或 `SSE`（紫色）
2. **URL** — 等宽字体，自动换行
3. **超时** — 如 `60 seconds`
4. **Headers**（有值时显示）— 键值对列表
   - 敏感请求头（名称含 auth/token/key/secret）自动脱敏：`aaa****bcde`
5. **级别** — 标签：`PROJECT` 或 `GLOBAL`（绿色）
6. **版本**（有值时显示）— 如 `my-server v1.0`

### 底部

- **🗑 删除服务器**：红色悬浮效果，需输入服务器名确认删除
- 仅 project 级别或管理员可见

---

## 编辑服务器

1. 元信息面板头部点击 ✏ 编辑按钮
2. 弹出同一表单，标题变为「Edit Server」
3. Name 字段不可修改
4. 修改配置后点击「Save Changes」保存（**无需重新测试**）

---

## 删除服务器

1. 元信息面板底部点击「删除服务器」
2. 确认对话框要求**输入服务器名称**才可删除：

```
┌─────────────────────────────────────┐
│ Delete MCP Server                    │
│                                     │
│ Delete server "my-server"?          │
│ This action cannot be undone.        │
│                                     │
│ Type the server name to confirm:     │
│ ┌─────────────────────────────────┐ │
│ │ my-server                        │ │
│ └─────────────────────────────────┘ │
│                                     │
│ [Cancel]                  [Delete]  │
└─────────────────────────────────────┘
```

**权限**：project 级别服务器所有用户可删除，global 级别仅管理员。

## 能力详情面板（中栏）

### 标签页

```
[TOOLS (5)]  [PROMPTS (2)]  [RESOURCES (1)]
```

每个标签显示对应项目数量，选中标签高亮。

### TOOLS（工具）

Agent 可调用的工具列表：

```
┌───────────────────────────────────────┐
│ mcp__my-server__search           [📋] │  ← 完整工具名（可复制）
│ 搜索文档并返回匹配结果                  │
├───────────────────────────────────────┤
│ mcp__my-server__translate        [📋] │
│ 将文本翻译为指定语言                    │
├───────────────────────────────────────┤
│ mcp__my-server__summarize        [📋] │
│ 生成文本摘要                           │
└───────────────────────────────────────┘
```

- 工具名格式：`mcp__{服务器名}__{工具名}`（等宽字体）
- 📋 复制按钮：复制完整工具名到剪贴板（点击后变 ✓，800ms 恢复）
- 点击工具行 → 右侧弹出工具测试抽屉
- 选中工具左侧显示蓝色边框

### PROMPTS（提示词）

服务器提供的提示词模板：
- 名称 + 描述
- 参数列表以标签形式展示

### RESOURCES（资源）

服务器提供的可访问资源：
- 名称 + URI（等宽字体）+ 描述
- 左侧绿色边框

### 错误状态

能力加载失败时显示错误信息和「Retry」按钮。

## 工具测试抽屉（右栏）

点击工具后从右侧滑入，宽度可拖拽调整（280px ~ 600px）。

### 头部

```
┌──────────────────────────────────────────────┐
│ mcp__my-server__search  [📋]  TOOL      [×] │
└──────────────────────────────────────────────┘
```

### 工具信息

- **描述**：工具用途说明
- **▸ INPUT SCHEMA**（可折叠）：点击展开 JSON 格式的输入参数定义，带复制按钮

### 测试表单

系统根据 input_schema 自动生成表单字段：

```
┌──────────────────────────────────────┐
│ TEST TOOL                            │
│                                      │
│ query *                    string    │  ← 必填（红色星号）
│ 搜索关键词                            │  ← 字段描述
│ ┌──────────────────────────────────┐ │
│ │ hello world                      │ │
│ └──────────────────────────────────┘ │
│                                      │
│ limit                      integer   │
│ 返回结果数量                          │
│ ┌──────────────────────────────────┐ │
│ │ 5                                │ │
│ └──────────────────────────────────┘ │
│                                      │
│ format                     enum      │
│ ┌──────────────────────────────────┐ │
│ │ json                          ▼  │ │  ← 枚举类型用下拉
│ └──────────────────────────────────┘ │
│                                      │
│ [▶ Run]                              │
└──────────────────────────────────────┘
```

**字段类型自动映射：**

| Schema 类型 | 输入控件 |
|------------|---------|
| string | 文本输入框 |
| number / integer | 数字输入框 |
| boolean | 下拉选择（true/false） |
| enum | 下拉选择（枚举值列表） |
| object / array | 多行文本框（输入 JSON） |

**验证规则：**
- 必填字段为空 → 红色边框 + 错误提示
- object/array 字段 JSON 格式错误 → 提示 Invalid JSON
- number/integer 字段非数字 → 提示 Must be a number
- 有错误时「Run」按钮禁用

### 测试结果

```
成功：
┌────────────────────────────────────┐
│ ▼ ✅ Success                  [📋] │  ← 可折叠，可复制全部结果
│                                    │
│ {                                  │  ← JSON 自动格式化
│   "results": [                     │
│     {"title": "文档1", ...},       │
│     {"title": "文档2", ...}        │
│   ]                                │
│ }                                  │
└────────────────────────────────────┘

失败：
┌────────────────────────────────────┐
│ ▸ ❌ Failed                   [📋] │
│   Error: Invalid query parameter   │
└────────────────────────────────────┘
```

**结果内容自动渲染：**
- **JSON**：自动检测并格式化显示
- **Markdown**：自动检测并渲染（含标题、列表、代码块等）
- **图片**：base64 图片直接渲染为 `<img>`
- **资源**：显示 RESOURCE 标签 + URI + 内容
- **纯文本**：等宽字体预格式化显示

## 在聊天中使用 MCP 工具

### MCP 选择策略

聊天输入框 + 按钮菜单中的 MCP 子菜单：

```
┌─────────────────────────┐
│ [Auto] [Disable]   2/3  │  ← 模式切换 + 选中数
│ ─────────────────────── │
│ ☑ api-server       HTTP │  ← 逐个勾选
│ ☑ search-tools     SSE  │
│ ☐ shared-docs      HTTP │
└─────────────────────────┘
```

| 策略 | 说明 |
|------|------|
| **Auto** | 自动使用所有已配置的服务器（默认），列表全选 |
| **Disable** | 禁用所有 MCP 工具，列表变暗 |
| **自选** | 点击任意复选框进入自选模式，显示选中数/总数 |

### 使用方式

Agent 根据对话内容自动判断是否调用 MCP 工具，无需手动指定：

```
用户：帮我搜索关于 React 的文档
Agent：（自动调用 MCP search 工具）找到以下相关文档...
```
